#### 情感关键词与灵感库

  - [情绪板：收集能体现目标情感的图片、音乐、文字]

#### 体验目标验证清单
- [ ] **一句话清晰性**：能否用一句话清楚传达游戏体验？

### 5.1.4 叙事载体层级设计 (Narrative Carrier Hierarchy)

#### 重度叙事 (打断操作体验)
- **CGI动画**：
  - 使用场景：片头、章节过渡、资料片
  - 资源要求：高，需要定制场景和高模
  - 节奏设计：缓-急-缓
- **过场动画 (Cutscene)**：
  - 使用场景：新角色引入、新场景展示、剧情预告
  - 资源分级：A类(重要剧情)、B类(一般剧情)、C类(简单剧情)
  - 跳过控制：首次不可跳过，后续可快进
- **对话镜头**：
  - 使用场景：关键NPC对话、情感表达
  - 机位设计：标准化机位分布
  - 交互设计：玩家可选择反馈

#### 中度叙事 (保留部分交互)
- **脚本事件**：
  - 类型：伪装变身、潜入、护送、跟随、搬运、载具飞行
  - 定制需求：特殊AI、场景事件、特效
- **面板对话**：
  - 展示形式：头像、半身像、全身像
  - 资源选择：2D、Live2D、3D模型
  - 功能类型：一般对话、分支对话、功能对话、交付对话
- **观察镜头**：
  - 目标物件：线索、场景细节、指引方向
  - 交互要求：明确的进入/退出提示
  - 机位处理：保持与游戏镜头的一致性

#### 轻度叙事 (不干扰操作)
- **环境叙事**：
  - 可探索物：场景中的交互元素
  - 探索提示：发光效果、特殊道具技能
  - 奖励机制：彩蛋奖励、收集进度
- **旁白与补充对白**：
  - 旅行对白：填补区域移动的无聊时间
  - NPC旁白：根据玩家数据动态调整内容
  - 旁白使用：仅在特殊世界观下有限使用
- **界面滤镜**：
  - 应用场景：回忆杀、透视观察、艺术风格化
  - 功能作用：增强沉浸感、指引行为、调整压力

### 5.1.3 Game Progression & Narrative Integration（剧情与游戏进程）

#### 5.1.3.1 剧情连贯性评估体系 (Narrative Coherence Evaluation System) [权重：10%]

##### 连贯性维度定义
- **时间连贯性** (Temporal Coherence)
  - 时间线一致性：事件发生的时间逻辑
  - 因果关系：前后事件的因果链条
  - 时代背景：历史设定与剧情的统一
- **角色连贯性** (Character Coherence)  
  - 性格一致性：角色行为与性格的匹配
  - 成长轨迹：角色发展的合理渐进
  - 关系发展：角色关系变化的自然性
- **世界观连贯性** (World-building Coherence)
  - 设定一致性：世界观规则的应用统一
  - 逻辑自洽：世界内部逻辑的无矛盾
  - 细节呼应：前后剧情细节的呼应

##### 连贯性评估清单
- [ ] **剧情逻辑**：主线剧情发展的逻辑合理性
- [ ] **角色动机**：角色行为动机的可信度
- [ ] **情感曲线**：剧情情感起伏的自然流畅
- [ ] **伏笔回收**：前期伏笔的合理回收
- [ ] **分支设计**：分支选择的逻辑合理性

#### 5.1.3.2 剧情结构设计模板 (Story Structure Template)

##### 三幕式结构框架
```
第一幕：设定 (Setup) - 占比25%
- 世界介绍：世界观和背景设定
- 角色登场：主要角色的初次亮相
- 矛盾建立：核心冲突的初步展现
- 激励事件：推动剧情发展的关键事件

第二幕：对抗 (Confrontation) - 占比50%  
- 上升行动：矛盾逐步激化
- 中点转折：剧情的重大转折点
- 障碍升级：困难和挑战的增加
- 谷底时刻：主角的最低谷时期

第三幕：解决 (Resolution) - 占比25%
- 高潮对决：最终冲突的爆发
- 下降行动：高潮后的余波处理
- 结局收尾：所有矛盾的最终解决
- 新平衡：故事结束后的新状态
```

##### 角色弧光设计
```
初始状态 → 触发事件 → 挣扎成长 → 关键选择 → 最终蜕变
    ↓           ↓           ↓           ↓           ↓
[角色缺陷] [生活变化] [面对挑战] [价值抉择] [人格升华]
```

##### 情节连贯性检查
```
时间线检查：
- 事件A发生 → 事件B发展 → 事件C解决
- 每个事件的时间间隔合理性
- 角色年龄和时代背景的一致性

因果关系检查：
- 原因A → 结果B → 影响C
- 每个结果都有明确的原因
- 避免机械降神式的解决方案
```

#### 5.1.3.3 对话连贯性设计 (Dialogue Coherence Design)

##### 对话逻辑框架
```
对话触发 → 信息传递 → 情感表达 → 选择分支 → 结果影响
    ↓           ↓           ↓           ↓           ↓
[触发条件] [核心信息] [情感基调] [玩家选择] [剧情走向]
```

##### 对话一致性检查
- **语言风格**：角色对话风格的统一性
- **知识水平**：角色对话内容的知识背景
- **情感状态**：对话情感与角色状态的匹配
- **信息递进**：对话信息的逐步展开逻辑

##### 分支对话设计
```
分支结构设计：
- 主线分支：影响剧情走向的关键选择
- 角色分支：影响角色关系的发展
- 信息分支：获得额外信息的探索选择
- 情感分支：影响情感氛围的偏好选择

连贯性保证：
- 所有分支都符合角色性格
- 分支结果有合理的逻辑解释
- 重要信息在所有分支中都能获得
```


#### 5.1.3.5 连贯性测试方法 (Coherence Testing Methods)

##### 专业评审流程
```
第一轮：策划内部评审
- 剧情逻辑检查
- 角色一致性验证
- 世界观统一性确认

第二轮：跨部门评审  
- 美术风格一致性
- 程序实现可行性
- 音效配合合理性

第三轮：外部评审
- 资深策划专家评审
- 目标用户测试
- 文学顾问评估
```

##### 玩家测试方法
```
测试指标设计：
- 理解度：玩家对剧情的理解程度 > 85%
- 沉浸感：玩家的情感投入程度 > 80%
- 合理性：玩家对剧情逻辑的认可度 > 75%
- 满意度：玩家对剧情体验的满意度 > 80%

测试问题设计：
- "你觉得这个角色的行为合理吗？为什么？"
- "剧情发展中有没有让你觉得困惑的地方？"
- "你最期待接下来发生什么？"
- "如果让你改变剧情，你会怎么改？"
```

#### 5.1.3.6 连贯性评分标准 (Coherence Scoring Standards)

##### 评分维度权重
- **时间逻辑**：30% - 时间线和因果关系的合理性
- **角色发展**：25% - 角性格弧和动机的一致性  
- **世界观**：20% - 世界设定的统一性和逻辑性
- **对话质量**：15% - 对话的自然性和信息传递
- **情感共鸣**：10% - 情感曲线的流畅性和感染力

##### 评分等级定义
```
90-100分（优秀）：
- 剧情逻辑完全自洽，无明显漏洞
- 角色发展自然合理，动机清晰可信
- 世界观设定统一，细节处理到位
- 对话自然流畅，信息传递有效
- 情感曲线设计合理，共鸣感强

80-89分（良好）：
- 剧情逻辑基本合理，存在细微优化空间
- 角色发展较为自然，个别行为可优化
- 世界观设定基本统一，少量细节需完善
- 对话较为自然，信息传递基本有效
- 情感曲线设计合理，有一定共鸣感

70-79分（合格）：
- 剧情逻辑存在小问题，需要调整
- 角色发展存在不合理之处
- 世界观设定存在轻微矛盾
- 对话存在生硬之处
- 情感曲线设计存在改进空间

60-69分（需改进）：
- 剧情逻辑存在明显问题
- 角色发展缺乏合理性
- 世界观设定存在矛盾
- 对话质量较差
- 情感曲线设计不合理

<60分（不合格）：
- 剧情逻辑严重混乱
- 角色发展完全不合理
- 世界观设定矛盾重重
- 对话质量极差
- 情感曲线设计失败
```
