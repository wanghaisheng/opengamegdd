##### 任务分类体系
- **主线任务** (Main Quests)

#### 4.2.6.4 商业系统架构 (Monetization System Architecture) [权重：3%]

##### 商业化分类体系

  - 宝箱系统：道具的随机开出
  - 扭蛋机制：低概率高价值的激励机制

  - 激励视频：主动观看获得奖励
  - 插屏广告：强制观看的广告展示
  - 交叉推广：其他游戏的推广展示

##### 用户分层设计体系
###### 用户分层定义
- **非R用户** (Non-Paying Users)
  - 定义：不充钱的玩家，占用户总量80%
  - 目标：保证留存，提高付费转化率
  - 策略：基础体验优化，长期留存设计
- **小R用户** (Light Payers)
  - 定义：充10-100元的玩家
  - 目标：持续小额付费，培养付费习惯
  - 策略：超值小额礼包，限购机制设计
- **中R用户** (Medium Payers)
  - 定义：充100-500元的玩家
  - 目标：稳定付费贡献，长期留存维护
  - 策略：超值礼包管理，实在内容提供
- **大R用户** (Heavy Payers)
  - 定义：充500元以上的玩家
  - 目标：最大化收入贡献，身份地位满足
  - 策略：身份象征，酷炫外显，专属服务

###### 分层体验设计
- **非R用户体验**：
  - 基础功能完全免费使用
  - 通过时间投入可获得大部分内容
  - 广告观看作为付费替代方案
  - 社交功能无限制使用
- **小R用户体验**：
  - 小额礼包显著提升游戏体验
  - 限购设计保证长期付费意愿
  - 性价比极高的入门级付费内容
  - 付费后获得明显的身份标识
- **中R用户体验**：
  - 可购买大部分超值礼包内容
  - 实在的游戏内容和能力提升
  - 不能快速售卖，需要时间积累
  - 专属客服和优先服务
- **大R用户体验**：
  - 所有付费内容均可购买
  - 身份地位象征和酷炫外显
  - 专属VIP服务和特权
  - 影响游戏发展的特殊权限

###### 分层付费策略
- **非R用户转化策略**：
  - 首充礼包：极低价格高价值
  - 限时特惠：制造紧迫感
  - 社交压力：付费用户特权展示
  - 体验限制：付费内容试用
- **小R用户留存策略**：
  - 连续充值奖励：培养付费习惯
  - 月卡订阅：稳定收入来源
  - 成长基金：长期投资回报
  - 专属活动：小额付费专属内容
- **中R用户维护策略**：
  - 高级礼包：针对性内容设计
  - 私人客服：一对一服务
  - 优先体验：新内容提前体验
  - 定制服务：个性化需求满足
- **大R用户服务策略**：
  - 专属管家：7×24小时服务

#### 4.2.6.6 经济平衡验证 (Economy Balance Validation)

##### 量化评估指标
- **付费率**：付费用户数 / 活跃用户数，目标5-15%
- **ARPU**：平均每用户收入，与品类基准对比
- **ARPPU**：平均每付费用户收入，控制在合理范围
- **LTV**：用户生命周期价值，大于CAC的3倍
- **付费渗透**：各付费档位的用户分布合理性

##### 分层用户分析指标
- **非R用户占比**：目标80%，留存率>60%
- **小R用户占比**：目标15%，月均付费10-100元
- **中R用户占比**：目标4%，月均付费100-500元
- **大R用户占比**：目标1%，月均付费500元以上
- **分层转化率**：非R→小R 8%，小R→中R 20%，中R→大R 15%

##### 分层收益贡献分析
```
收益贡献模型：
- 非R用户：广告收益 + 社交价值，占总收益5-10%
- 小R用户：小额付费稳定贡献，占总收益20-30%
- 中R用户：主要付费群体，占总收益40-50%
- 大R用户：核心收入来源，占总收益20-30%

健康指标：
- 大R用户收益占比不超过50%（避免过度依赖）
- 分层用户结构金字塔分布合理
- 各层用户付费深度与游戏时长正相关
```

##### 价格弹性测试
``` 
分层价格敏感度测试：
- 非R用户：首充礼包 $0.99 → $1.99，转化率变化
- 小R用户：月卡 $4.99 → $9.99，续费率变化
- 中R用户：成长基金 $29.99 → $49.99，购买率变化
- 大R用户：至尊礼包 $99.99 → $199.99，购买率变化

最优价格策略：
收益 = 价格 × 转化率 × 用户基数
考虑长期留存和用户升级路径
```

##### A/B测试框架
- **分层测试**：不同用户群体的付费反应
- **价格测试**：不同价格档位的分层转化效果
- **包装测试**：不同商品包装对各层用户的吸引力
- **时机测试**：不同付费时机对各层用户的影响
- **渠道测试**：不同付费渠道的分层效果对比

##### 概率设计逻辑

```
- 300抽必得指定SSR

```

- **方差控制**：极端情况的概率控制
- **保底机制**：防止极端非酋体验
- **概率透明**：符合法规要求的概率公示

#### 4.2.6.8 广告变现设计 (Ad Monetization Design)

##### 激励视频逻辑
```
触发场景设计：
- 体力不足：观看广告获得体力
- 失败复活：观看广告继续游戏
- 奖励加倍：观看广告双倍奖励
- 宝箱加速：观看广告立即开启

价值平衡：
- 广告价值：$0.02-0.05
- 奖励价值：$0.01-0.03
- 平台分成：60% 给开发者
```

##### 广告频率控制
- **频率限制**：同一用户每小时最多观看10次
- **体验保护**：关键游戏流程不插入广告
- **用户选择**：提供付费去广告选项
- **地域适配**：不同地区广告策略差异化

##### 逻辑性评分标准
- **90-100分**：商业化逻辑完美，用户体验与收益平衡
- **80-89分**：商业化逻辑合理，存在优化空间
- **70-79分**：商业化存在小问题，需要调整
- **60-69分**：商业化存在明显问题，影响体验
- **<60分**：商业化设计严重缺陷，不建议采用

##### 数据监控指标
- **转化漏斗**：各层级转化率监控

- **合规监控**：法规符合性检查

```
非R用户监控：

- 内容消费：游戏内容完成率、社交互动频率

小R用户监控：
- 付费频率：月均付费次数、付费间隔

- 流失风险：付费间隔延长、活跃度下降预警

中R用户监控：
- 付费深度：月均付费金额、付费上限接近度

大R用户监控：

```

##### 分层用户预警机制
```
流失预警：
- 非R用户：连续7日未登录，留存率下降
- 小R用户：连续30日未付费，付费间隔延长
- 中R用户：连续60日付费下降，满意度降低
- 大R用户：连续90日付费减少，活跃度异常

升级预警：
- 非R→小R：达到付费门槛，付费意愿增强
- 小R→中R：小额付费稳定，有升级潜力
- 中R→大R：付费能力提升，需要特殊服务

健康度预警：
- 分层结构失衡：某层用户占比异常
- 收益过度依赖：大R用户收益占比>50%
- 转化路径异常：某层转化率异常变化
```

##### 游戏类型选择指南

###### 如何确定游戏类型
1. **核心玩法分析**：确定游戏最核心的玩法机制
2. **目标用户分析**：明确主要用户群体的偏好
3. **市场定位分析**：分析在市场中的定位和差异化
4. **技术实现分析**：评估技术实现的可行性

###### 混合类型处理
对于混合类型的游戏，按照以下原则处理：
- **主次分明**：以占比最高的类型为主要框架
- **机制融合**：将次要类型的特色机制融入主框架
- **体验统一**：确保不同机制的体验协调统一
- **用户匹配**：满足目标用户的复合需求

#### 4.2.7.2 核心数值公式 (Core Numerical Formulas)

##### 战斗公式体系
```
伤害计算公式：
FinalDamage = BaseDamage × (1 + AttackBonus) × (1 + CritDamage) × ElementBonus 
             - TargetDefense × (1 - ArmorPenetration) × (1 - Resistance)

治疗计算公式：
FinalHeal = BaseHeal × (1 + HealPower) × (1 + TargetHealReceived) 
           × (1 - HealReduction)

状态效果公式：
StatusDamage = BaseStatusDamage × StackCount × Duration 
               × (1 + StatusAmplification)
```

##### 成长公式体系
```
等级经验公式：
ExpNeeded = BaseExp × (Level ^ GrowthFactor) × ExpMultiplier

属性成长公式：
FinalStat = BaseStat + (Level × GrowthPerLevel) 
            + EquipmentBonus + BuffBonus

战力计算公式：
PowerScore = HP_Weight × HP + Attack_Weight × Attack 
            + Defense_Weight × Defense + Skill_Weight × SkillPower
```

##### 经济公式体系
```
资源产出公式：
ResourceIncome = BaseIncome × (1 + EfficiencyBonus) 
                 × (1 + VIPBonus) × ActivityMultiplier

资源消耗公式：
ResourceCost = BaseCost × (Level ^ CostGrowth) 
               × (1 - CostReduction)

兑换比例公式：
ExchangeRate = BaseRate × MarketDemand × SupplyDemand
```
